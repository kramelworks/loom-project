# Loom Governance Model

## Governance Principles

Loom is an Open Source project maintained by Kramelworks Academy.

The project follows these principles:

- transparency
- collaborative learning
- code quality
- merit-based contribution
- respectful communication

---

# Roles and Responsibilities

## 1. Product Owner

Responsible for:
- product vision
- roadmap
- prioritization
- final product decisions

Initial owner:
Kramelworks

---

## 2. Core Maintainers

Responsible for:
- architecture decisions
- reviewing major PRs
- approving RFCs
- release management

Permissions:
- full repository access

---

## 3. Tech Leads

Areas:
- Backend
- Frontend
- DevOps

Responsible for:
- technical guidance
- code reviews
- mentoring contributors

Permissions:
- write access

---

## 4. Mentors

Responsible for:
- onboarding
- helping contributors
- technical support

Permissions:
- triage access

---

## 5. Contributors

Responsible for:
- implementing issues
- opening PRs
- participating in discussions

Permissions:
- fork + pull request

---

## 6. Trainees

Responsible for:
- learning
- completing academy tasks
- submitting supervised contributions

Permissions:
- fork + pull request

---

# Decision Process

Small decisions:
- Maintainers decide

Architecture changes:
- RFC required

Breaking changes:
- Product Owner approval required

---

# Communication Channels

Primary:
- GitHub Issues
- GitHub Discussions
- Pull Requests

Secondary:
- Discord (future)

---

# Promotion Model

Trainee
→ Contributor
→ Mentor
→ Tech Lead
→ Core Maintainer

Promotion is merit-based.
